Statements

Statements are syntax constructs and commands that perform actions.

The code can have as many statements that you can need. Statements can be separated with a semicolon.

FOR EXAMPLE

Here is a split of "Hello World" into two alerts ⤵️

alert('Hello');alert('Wolrd');

Usually, statements are written on separate lines to make the code more readable:

alert('alert');
alert('World');

Semicolons

A semicolon can be omitted in most cases when a line break exists.

This would also work to ⤵️

alert('alert')
alert('World')

Here, Javascript interprets the line break as an "implicit" semicolon . This is called an automatic semicolon insertion.

In most cases, a new line implies a semicolon. But it's not always.

FOR EXAMPLE ⤵️

alert('3 +
1
+ 2');

Is recommend putting semicolons between statements even if they are separated by newlines. This rule is widely adopted by the community. Let’s note once again – it is possible to leave out semicolons most of the time. But it’s safer to use them.

Comments

As times go on, programs become more and more complex. It becomes necessary to add comments which describe what the code does and why.

Comments can be put into any place of a script. They don’t affect its execution because the engine simply ignores them.

One-line comments start with two forward slash characters //.

The rest of the line is a comment. It may occupy a full line of its own or follow a statement.

Like here ⤵️

// This comment occupies a line of its own
alert('alert');
alert('World'); // This comment follows the statement

Multi-line comments start with a forward slash and an asterisk /* and end with an asterisk and a forward slash */.

Like this ⤵️

/* An example with two messages.
This is a multiline comment.
*/
alert('Hello');
alert('World');
